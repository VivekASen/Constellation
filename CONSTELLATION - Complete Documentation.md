# 📚 CONSTELLATION - Complete Documentation

## 🎯 Project Overview

**Constellation** is an AI-powered iOS app that helps users discover and connect ideas across all their media consumption: movies, TV shows, podcasts, articles, and books.

**Core Value Proposition:**
> "Your AI research assistant for everything you watch, read, and listen to"

### The Problem It Solves

People consume content across multiple platforms:
- Movies on Netflix, theaters, Letterboxd
- TV shows on streaming services
- Podcasts on Spotify, Apple Podcasts
- Articles saved in browser, Pocket, Readwise
- Books tracked on Goodreads

**But:**
- No way to see thematic connections across these
- Hard to remember "I watched a movie about this topic"
- Discovery is siloed (Netflix only knows Netflix)
- Research is manual ("What did I consume about the Kennedys?")

### What Constellation Does

1. **Unified Tracking** - Track all media in one place
2. **AI Theme Extraction** - Automatically tags content with themes
3. **Cross-Media Discovery** - "Show me everything about space exploration"
4. **Smart Recommendations** - Based on themes, not just genres
5. **Connection Finding** - "These share a director/theme/topic"
6. **Research Mode** - Deep dives into topics across formats

## 🤖 AI Services

### 1. ThemeExtractor

**Purpose:** Extract themes from media content
**Model:** Llama 3.2 (local via Ollama)
**Input:** Movie metadata (title, overview, genres, user notes)
**Output:** Array of theme strings (e.g., `["time-travel", "sci-fi", "paradox"]`)

**How it works:**
```swift
let themes = await ThemeExtractor.shared.extractThemes(from: movie)
// Returns: ["space-exploration", "survival", "isolation"]
```

**Prompt structure:**
- System: Knowledge management assistant
- User: Movie details + instructions to extract 3-7 themes
- Format: Lowercase, dash-separated (e.g., "coming-of-age")

### 2. DiscoveryEngine

**Purpose:** Smart content discovery based on interests
**Model:** Llama 3.2 for understanding, TMDB API for recommendations
**Input:** User query (e.g., "space exploration")
**Output:** `DiscoveryResult` with matches + recommendations + questions

**How it works:**

**Step 1: Understand Query**
```swift
// User searches: "space exploration"
// AI returns:
{
  themes: ["space", "exploration", "astronauts"],
  genres: ["sci-fi", "documentary"],
  mood: "wonder and discovery",
  isGenre: false,
  suggestions: ["Interstellar", "The Martian", "Apollo 13"]
}
```

**Step 2: Smart Recommendations**
- Searches TMDB using AI suggestions (not literal query)
- "space exploration" → searches "Interstellar", "The Martian", etc.
- Returns popular, highly-rated movies
- Filters out movies user already has

**Step 3: Find Library Matches**
- Searches user's library by themes, genres, titles
- Uses AI understanding, not literal string matching

**Step 4: Generate Follow-Up Questions**
- "What format?" → Movies, TV, Documentaries
- "What vibe?" → Action-packed, Thoughtful, etc.
- "More like X?" → Find similar content

### 3. SynthesisEngine (TODO - Future)

**Purpose:** Generate insights about collections
**Input:** Collection of related items
**Output:** AI-written summary connecting themes

Example:
> "These 5 movies explore decision-making under pressure. From 12 Angry Men's deliberative process to Moneyball's data-driven approach, they show different frameworks for making high-stakes choices..."

## 🔧 Setup & Configuration

### Prerequisites

1. **Xcode 15+**
2. **iOS 17.0+ device/simulator**
3. **Ollama installed** (for local AI)
4. **TMDB API key**

### Installation Steps

**1. Clone & Open:**
```bash
cd ~/development/constellation-app
open Constellation.xcodeproj
```

**2. Configure TMDB:**
- Get API key: https://www.themoviedb.org/settings/api
- Add to `Services/API/TMDBService.swift` line 6
- Replace `"YOUR_API_KEY_HERE"` with actual key

**3. Start Ollama:**
```bash
# Install Ollama
brew install ollama

# Start server
ollama serve

# Pull model (in another terminal)
ollama pull llama3.2

# Test
ollama run llama3.2 "Hello"
```

**4. Build & Run:**
- Select target device/simulator
- Cmd + R to build and run

### Common Issues

**"No API key" error:**
- Check `TMDBService.swift` has valid key
- Test key at: https://api.themoviedb.org/3/movie/550?api_key=YOUR_KEY

**"Ollama error":**
- Make sure `ollama serve` is running
- Check model is installed: `ollama list`
- Should see `llama3.2` in the list

**"Theme extraction stuck":**
- Themes extract in background (5-10 seconds)
- Pull to refresh or reopen movie to see themes
- Check Ollama terminal for errors

## 💡 Key Design Decisions

### Why Local-First (SwiftData)?

- Privacy: All data stays on device
- Performance: Instant access, no network
- Offline: Works without internet
- Cost: No backend to maintain

### Why Ollama (Local AI)?

- Privacy: Themes extracted on-device
- Cost: Free, no API fees
- Speed: Fast with small models (llama3.2)
- Offline: Works without internet

**Trade-off:** Slower than cloud APIs, requires local compute

### Why TMDB?

- Free API with generous limits
- Best movie/TV database
- High-quality posters
- Active development

### Why SwiftUI + SwiftData?

- Modern iOS development
- Declarative UI (faster to build)
- Native performance
- Built-in persistence

## 📝 Code Conventions

**Naming:**
- Models: `Movie`, `Theme`, `ItemCollection`
- Views: `MovieDetailView`, `DiscoveryView`
- Services: `ThemeExtractor`, `TMDBService`

**Structure:**
- Models use `@Model` macro (SwiftData)
- Views use `@Query` for data access
- Services are singletons (`.shared`)

**AI Prompts:**
- Always specify output format
- Use examples in prompts
- Request lowercase, dash-separated themes

**Async/Await:**
- All API calls are async
- Use `Task {}` for background work
- Show loading states in UI

### Debugging AI Issues

**Ollama not responding:**
```bash
# Check if running
ps aux | grep ollama

# Restart
killall ollama
ollama serve
```

**Themes not extracting:**
- Check Ollama terminal for errors
- Test prompt directly: `ollama run llama3.2 "Extract themes..."`
- Verify model name in code matches installed model

**Bad theme quality:**
- Improve prompt in `ThemeExtractor.swift`
- Add more examples
- Try different temperature (currently 0.3)
